# ProofMarket Deployment

ProofMarket is a static Vite app. It can be deployed without a backend, wallet, paid API, login, or hidden secret because the public judge path uses synthetic replay fixtures.

## Required Pre-Deployment Check

Run:

```bash
npm install
npm test
npm run lint
npm run build
npm audit --audit-level=moderate
```

Confirm `dist/` is generated by the build and is not committed.

## Visual Polish Gate

Before recording or submitting:

1. Open the deployed or local app on desktop width.
2. Confirm the first viewport reads like a product demo, not a developer console:
   - `ProofMarket` title is visible.
   - `Replay late-goal settlement` is the dominant action.
   - Verdict strip clearly shows `PREVIOUS RECEIPT 1-1 -> TXLINE PROOF VERIFIED -> FINAL SETTLEMENT 2-1`.
   - Compliance is visible but does not dominate the first screen.
3. Open or emulate a mobile viewport around 390px wide.
4. Confirm there is no horizontal scroll and the verdict strip stacks cleanly.
5. Click the CTA once on both desktop and mobile and confirm the proof lifecycle and ledger remain readable.

## Environment Variables

For the replay-first public deployment, no environment variables are required.

Optional documented values:

```bash
TXLINE_API_BASE=https://txline.txodds.com
TXLINE_API_TOKEN=
TXLINE_MODE=replay
SOLANA_RPC_URL=https://api.devnet.solana.com
NEXT_PUBLIC_APP_MODE=replay
NEXT_PUBLIC_TRACK_ID=proofmarket
PROOFMARKET_PROGRAM_ID=
```

Do not deploy real TxLINE tokens, private keys, wallet seed phrases, or sponsor-private material.

## Vercel

1. Import the public GitHub repo into Vercel.
2. Framework preset: `Vite`.
3. Build command: `npm run build`.
4. Output directory: `dist`.
5. Leave secrets empty for the replay-first judge path.
6. Deploy and copy the public URL into `README.md` and `SUBMISSION.md`.

## Netlify

1. Import the public GitHub repo into Netlify.
2. Build command: `npm run build`.
3. Publish directory: `dist`.
4. Leave secrets empty for the replay-first judge path.
5. Deploy and copy the public URL into `README.md` and `SUBMISSION.md`.

## GitHub Pages

1. Build locally with `npm run build`.
2. Use a Pages workflow or static-site action that publishes `dist/`.
3. Keep `.env`, tokens, private keys, and `node_modules/` out of the repo.
4. Copy the Pages URL into `README.md` and `SUBMISSION.md`.

## Judge Path After Deploy

1. Open the public app URL.
2. Confirm the banner says `Replay demo mode`.
3. Click `Replay late-goal settlement`.
4. Verify the verdict strip moves from `PREVIOUS RECEIPT 1-1` to `TXLINE PROOF VERIFIED` to `FINAL SETTLEMENT 2-1`.
5. Review the receipt diff and simulated ledger.

## Deployment Limit

This deployment does not claim live TxLINE production data is wired in. It is a replay-first public submission with documented live endpoint targets for the next adapter step.
